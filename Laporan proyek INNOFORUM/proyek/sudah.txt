Next Step: Reporting/Flagging
Diskusi & Rencana Fitur Reporting/Flagging
A. Tujuan
	User bisa melaporkan (report/flag) thread (pertanyaan) atau komentar yang dianggap melanggar aturan (spam, SARA, dsb). Laporan akan masuk ke admin untuk ditindaklanjuti.
B. Komponen Fitur
	Tombol "Laporkan"
		Ada di setiap thread dan/atau komentar.
		User yang login bisa klik tombol tersebut.
	Form laporan
		User memilih alasan laporan (misal: Spam, SARA, Iklan, Lainnya).
		Bisa isi deskripsi tambahan.
	Penyimpanan laporan
		Database tabel "reports" (menyimpan: pelapor, tipe (thread/komentar), id target, alasan, deskripsi, status laporan).
		Notifikasi/admin panel
		Admin bisa lihat daftar laporan, dan update status laporan (selesai, diabaikan, valid, dsb).
		(Opsional) User pelapor dapat info update laporan.
C. Flow
	User klik "Laporkan" → form modal/pop-up → submit → data masuk tabel "reports".
	Admin login → menu "Laporan" → lihat & kelola laporan.
































1. Alasan Laporan
Berikut contoh alasan laporan yang sering dipakai di forum:
	Spam (iklan, promosi, dsb)
	SARA (Suku, Agama, Ras, dan Antar Golongan)
	Iklan (promosi produk, jasa, dsb)
	Kata-kata kasar (kata tidak sopan, penghinaan, bullying, dsb)
	Lainnya (alasan lain, user bisa isi sendiri detailnya)
	Contoh pada form:
		Code
		[ ] Spam
		[ ] SARA
		[ ] Iklan
		[ ] Kata-kata kasar
		[ ] Lainnya: [____________________]
2. Siapa yang Bisa Melapor
	Semua user yang login bisa melaporkan (role apapun: mahasiswa, dosen, admin, dst).
3. Siapa yang Bisa Melihat Laporan
	Hanya admin yang bisa melihat laporan, memproses/mengubah status laporan.
	User biasa tidak bisa melihat laporan orang lain.
4. Notifikasi Update Status ke User Pelapor (Penjelasan)
	Yang dimaksud notifikasi update status ke user pelapor adalah:
	User pelapor (misal: User B) akan mendapat pemberitahuan jika laporan yang dia buat ada update dari admin.
		Contoh: Setelah admin cek, status laporan berubah dari "Menunggu" menjadi "Diproses", atau "Diterima", atau "Ditolak".
		Notifikasi bisa berupa:
		Pesan di sistem (“Laporan Anda telah diproses admin”),
		Email (jika perlu),
		Atau badge/icon di menu user (“Ada pembaruan pada laporan Anda”).

	User yang melaporkan akan tahu jika laporannya sudah dicek admin.
	User yang dilaporkan tidak otomatis tahu, kecuali admin memberikan sanksi/teguran secara terpisah.

Kesimpulan
Semua user bisa melapor.
Hanya admin yang bisa melihat laporan.
Notifikasi update status bersifat opsional kepada user pelapor, bukan ke user yang dilaporkan.
